俗话说:基础不牢，地动山摇.每个程序员CRUD都是必备操作.第一篇文章就复习CRUD吧!

1.创建数据源,连接数据库

首先用Navicat随便建一个库.然后写配置文件连接到这个数据库.

2.取数逻辑

传统的页面取后台数据。需要些 controller,manager,dao 等一整套逻辑。

model

创建一个对象来对应数据库各个属性.

public class OaStudent {
    private String ID;
    private String NAME;
    private String SEX;

    @Override
    public String toString() {
        return "OaStudent{" + "ID='" + ID + '\'' + ", NAME='" + NAME + '\'' + ", SEX='" + SEX + '\'' + '}';
    }

    public String getID() {
        return ID;
    }

    public void setID(String ID) {
        this.ID = ID;
    }
    public String getNAME() {
        return NAME;
    }
    public void setNAME(String NAME) {
        this.NAME = NAME;
    }

    public String getSEX() {
        return SEX;
    }

    public void setSEX(String SEX) {
        this.SEX = SEX;
    }
}

Dao层

首先创建一个接口,写上CRUD四个方法.(为了便于多态)

public interface OaStudentDao {
    public OaStudent insert(OaStudent student);
    public String delete(String id);
    public OaStudent update(OaStudent student);
    public List<OaStudent> find(OaStudent student);
}

随后写这个接口的实例,并且继承MyBatisDaoImpl.

@Repository
public class OaStudentDaoImp extends MyBatisDaoImpl implements OaStudentDao {
    @Resource
    private JdbcTemplate jdbcTemplate;

    static String DZZWLSY_NAME="LSY_GOVENGINE_LAND";

    @Override
    public OaStudent insert(OaStudent student) {
         sqlSessionTemplate.selectList("insert1",student);
        return student;
    }

    @Override
    public String delete(String id) {
         sqlSessionTemplate.selectList("delete1",id);
        return id;
    }

    @Override
    public OaStudent update(OaStudent student) {
         sqlSessionTemplate.selectList("update1",student);
        return student;
    }

    @Override
    public List<OaStudent> find(OaStudent student) {

        return sqlSessionTemplate.selectList("findsex",student);
    }
    @Override
    public String getNamespace() {
        return OaStudentDaoImp.class.getName();
    }
}
